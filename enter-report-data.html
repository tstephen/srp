1. Enter Report Data

1.1. Actor

End user of the system, typically an employee of a reporting organisation

1.2. User story

The user will be able to enter the same data as currently requested in the spreadsheet. Exact nature and structure of this data is described in the data model.

The user may enter one yearâ€™s data in a single page. All entries are automatically saved minimising the risk of any being lost. It is possible to scroll to previous and subsequent years, providing they exist. Only one year into the future will be available.

None of this data is shared with the SDU until it is submitted by a suitably qualified user within the reporting organisation. Suitably qualified means a user with the Approver role.

1.3. User outcome

The user may login and change any data (s)he wishes until prior to the report being submitted. After that it may only be viewed, or copied to created a revised submission. See here for details of that.
