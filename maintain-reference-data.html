1. Maintain Reference Data

1.1. Actor

SDU analyst or System administrator

Requires admin role.

1.2. User story

Several pieces of data are expected to change over time and will dramatically affect the calculations reported. For example the Carbon intensity of certain operations will change as the wider economy is gradually de-carbonised.

Once logged in the user will the available application pages including any maintenance pages. Navigating to the appropriate one, the user will be able to enter or update the appropriate values.

Most, perhaps all, reference data will have an applicability period so there should not be any risk of confusion due to reports changing after being submitted so long as reference data is defined ahead of the reporting period. Should retrospective changes need to be made appropriate communication to users should accompany it.

1.3. Exclusions

Note that all reference data will need to have values provided as part of the initial system implementation or the calculations will not work. Therefore it is not necessary to add new reference data but only to update the existing values.

1.4. User outcome

Reports should reflect the changed reference data on next viewing.
